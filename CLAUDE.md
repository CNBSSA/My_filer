## 1. North Star (do not weaken, do not shrink)

**Mai Filer** is an **AI-native Nigerian tax e-filing platform** — a peer to TurboTax
and TaxSlayer, but born in the age of AI. Conversation with Mai Filer **is** the
filing experience. Forms are generated by the agent from dialogue and documents;
they are not the primary interface.

- **Long-term**: live integration with the **Nigeria Revenue Service (NRS)** via the
  **Rev360** platform.
- **Short-term**: file online inside the app and generate downloadable,
  NRS-compliant filing packs for manual submission.
- **Always**: Mai Filer educates users, maximizes legal tax benefits, and prevents
  filing errors before submission.

Mai Filer is **not**:
- a form wizard with a chatbot on the side
- a batch-upload tool
- an FAQ bot
- a white-label of any existing product

## 2. Mai Filer's Ten BIG Roles (LOCKED)

These are the agent's responsibilities. They do not shrink. Each role is implemented
as Claude API tool use; Mai Filer (the orchestrator) delegates to specialized
sub-agents.

1. **Taxpayer Concierge** — primary chat face; profiles the user (PAYE / SME /
   freelancer / multi-income); builds a personalized filing plan.
2. **Document Intelligence** — ingests payslips, receipts, bank statements,
   contracts, CAC certificates, prior filings; extracts structured data via
   Claude Vision.
3. **Calculator & Optimizer** — runs PIT / CIT / VAT / WHT / PAYE / Development
   Levy math; explores reliefs, pension, NHIS, CRA to minimize liability legally.
4. **Compliance Advisor** — monitors the 24-hour MBS sync window, the ₦100m VAT
   threshold, the 55-field rule; flags risks proactively.
5. **Explanation Engine** — translates tax jargon to plain English (Hausa /
   Yoruba / Igbo are roadmapped, not v1).
6. **Audit Shield** — pre-filing review agent; catches missing fields, NIN-name
   mismatches, schema violations **before** NRS does.
7. **Filing Orchestrator** — short-term: generates UBL 3.0 / 55-field packs for
   download. Long-term: handles OAuth2 + HMAC-SHA256 handshake with NRS.
8. **Learning Partner** — remembers the user year-over-year, surfaces anomalies
   ("your VAT collection dropped 40% this quarter").
9. **Receipt & E-Invoice Co-Pilot** — for SMEs; generates MBS-compliant e-invoices
   with QR + CSID.
10. **Multi-Agent Orchestrator** — Mai does not do everything herself; she routes
    work to Calculator, Verifier, Filer, Reviewer sub-agents via tool use.

## 3. Locked Technology Stack

| Layer | Choice | Reason |
|---|---|---|
| Backend | **FastAPI** (Python 3.12) | async for real-time NRS sync |
| Frontend | **Next.js 15** + TypeScript + Tailwind | chat UI + dashboards |
| Database | **PostgreSQL 16** | ACID — tax data cannot glitch |
| Vector | **pgvector** | RAG over tax law + user memory |
| Queue | **Celery + Redis** | async NRS filing + retries |
| LLM | **Claude API** — Opus 4.7 (Mai reasoning), Sonnet 4.6 (tools), Haiku 4.5 (cheap tasks) | with prompt caching for tax-law context |
| Object storage | S3-compatible, **Nigerian-hosted** (Galaxy Backbone / Rack Centre) | NITDA data residency |
| Secrets | `.env` (dev) → Vault (prod) | never in code |
| Container | Docker + docker-compose | local dev parity |

Do not swap any of these without express approval.

## 4. Compliance Guardrails (NON-NEGOTIABLE)

- **NDPR / NDPC** — consent capture before every NIN query; annual DPCA audit; raw
  NIN never stored (hash + encrypted vault only).
- **NITDA** — software clearance; no backdoors; indigenous-content friendly.
- **NTAA 2026** — 24-hour real-time sync for medium/large taxpayers.
- **Data residency** — primary tax + biometric data must sit on servers inside
  Nigeria. AWS us-east / GCP Belgium are violations without a local proxy.
- **Identifiers** — NIN for individuals, CAC RC for businesses. These are the
  only valid primary keys.
- **E-invoice standard** — UBL 3.0, 55 mandatory fields across 8 sections,
  cryptographic stamp (QR + CSID + IRN) from NRS on every successful filing.
